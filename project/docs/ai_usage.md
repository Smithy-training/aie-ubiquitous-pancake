# Использование ИИ/ML в проекте

Проект использует классический ML-подход, а не генеративную LLM-модель. ИИ-компонент здесь - обученная модель машинного обучения, которая прогнозирует спрос товара по входным признакам.

## Где используется ML

ML используется в основном сценарии:

```text
POST /v1/predict
```

Входные признаки:

- `product_id` - идентификатор товара;
- `horizon_days` - горизонт прогноза в днях;
- `current_base_price` - текущая базовая цена;
- `is_weekend` - является ли день выходным.

Модель возвращает прогноз спроса:

```text
predicted_demand_units
```

## Какая модель используется

Текущая версия использует:

```text
RandomForestRegressor
```

В runtime модель не обучается. Сервис загружает готовые предобученные артефакты из `artifacts/`.

`scripts/train_model.py` используется только для воспроизведения демонстрационных артефактов на синтетическом датасете `synthetic-pricing-v1`.

Артефакты:

- `artifacts/model_v1.pkl` - sklearn pipeline с `DictVectorizer` и `RandomForestRegressor`;
- `artifacts/scaler_v1.pkl` - вспомогательная статистика цен;
- `artifacts/model_metadata.json` - версия, признаки, метрики, dataset version и checksum.

## Что делает модель

Модель решает задачу регрессии:

```text
features -> demand_units
```

То есть она оценивает, сколько единиц товара может быть востребовано при заданном товаре, цене, горизонте и признаке выходного дня.

## Что не является ML

Рекомендованная цена:

```text
recommended_price
```

рассчитывается отдельной бизнес-логикой в `src/app/services/model_loader.py`. Она использует прогноз спроса модели, но сама по себе не является прямым output модели.

Это сделано намеренно: в реальных сервисах ML-модель часто прогнозирует один показатель, а финальное бизнес-решение получается дополнительной логикой, ограничениями и правилами.

## Confidence score

Поле:

```text
confidence_score
```

является инженерной оценкой уверенности сервиса. Оно учитывает:

- разброс предсказаний деревьев внутри Random Forest;
- насколько цена находится в диапазоне обучающих данных;
- горизонт прогноза.

Если confidence ниже порога `ml.min_confidence_threshold`, сервис возвращает warning.

## Explain endpoint

Endpoint:

```text
POST /v1/explain
```

возвращает prediction и список инженерных объяснений. Это не SHAP и не строгое feature attribution. В текущей версии объяснение эвристическое: сервис показывает, был ли товар известен модели, насколько длинный горизонт прогноза, находится ли цена в обучающем диапазоне и какой тип модели использован.

Такой формат полезен для демонстрации explainability-слоя вокруг ML-сервиса, но не заменяет полноценный анализ вклада признаков.

## Ограничения

Текущая модель демонстрационная:

- данные синтетические;
- нет реального товарного каталога;
- нет исторических продаж;
- нет внешних факторов вроде сезона, акций, остатков на складе или конкурентов;
- нет регулярного мониторинга drift на реальных данных.

Для production-версии нужно заменить синтетику на реальные данные, добавить feature store или стабильный пайплайн подготовки признаков, настроить регулярное переобучение и мониторинг качества модели.
