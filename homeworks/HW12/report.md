-   Стационарность: Вероятно нестационарный (есть тренд)

----------

## 4. Temporal Split

**Разделение:** Train 70% / Validation 15% / Test 15%

**Границы:**

-   Train: 1022 samples (2020-01-01 до 2022-10-18)
-   Validation: 219 samples (2022-10-19 до 2023-05-25)
-   Test: 220 samples (2023-05-26 до 2023-12-31)

**После создания признаков (удаление NaN от лагов):**

-   Train: 1012, Val: 217, Test: 218

**Почему random split некорректен:**

1.  Утечка данных из будущего в тренировку
2.  Временные зависимости нарушаются
3.  Модель будет учить паттерны, которых нет в продакшене
4.  Метрики валидации будут излишне оптимистичными

----------

## 5. Признаки

**Лаговые:** lag_1, lag_7, lag_14

**Rolling:** rolling_mean_7, rolling_std_7 (со shift(1) для избежания утечки)

**Календарные:** day_of_week, day_of_month, month, day_of_year

**Циклические:**

-   day_of_week_sin, day_of_week_cos
-   month_sin, month_cos

**Важно:** Все признаки строятся без утечки из будущего (shift(1) для rolling)

----------

## 6. Эксперименты

**B1 (naive-last):** Прогноз = последнее известное значение

**B2 (moving-average):** Прогноз = скользящее среднее (window=7)

**B3 (ridge-lag-features):** Ridge Regression (alpha=1.0) на лаговых + rolling + календарных признаках

**R1 (gru-forecast):** GRU (hidden=64, layers=2, dropout=0.2) на оконном представлении (window=14)

-   Batch size: 32
-   Learning rate: 1e-3
-   Epochs: 50
-   Optimizer: Adam
-   Scheduler: ReduceLROnPlateau (patience=5, factor=0.5)
-   Total parameters: 39,937

## 8. Анализ

**Baseline сравнение:**

-   B1 (Naive) — простейший бенчмарк, худший результат (MAE: 24.14)
-   B2 (MA) — значительно лучше B1, учитывает локальную динамику (MAE: 4.27)
-   B3 (Ridge) — лучший результат, использует лаги и календарные признаки (MAE: 4.22)
-   R1 (GRU) — хорошие результаты, но немного хуже Ridge на этом датасете (MAE: 5.14)

**GRU обучение:**

-   Сходимость за 50 эпох
-   Best validation loss: 0.120462
-   Validation loss стабилизируется к концу обучения
-   Early stopping можно добавить для предотвращения переобучения

**Почему Ridge лучше GRU в данном случае:**

-   Датасет относительно небольшой (1461 наблюдение)
-   Ridge быстрее обучается и требует меньше данных
-   Лаговые признаки хорошо捕捉ают зависимости в этом ряде
-   GRU требует больше данных для эффективного обучения

----------

## 9. Итоговый вывод

Для прогнозирования временных рядов **Ridge на лаговых признаках показал наилучшие результаты** на данном датасете. GRU показал хорошие результаты, но требует больше данных для эффективного обучения.

**Ключевые выводы:**

1.  Temporal split критически важен для временных рядов
2.  Лаговые признаки значительно улучшают качество прогноза
3.  Простые модели (Ridge) могут превзойти нейросети на небольших датасетах
4.  Масштабирование обязательно для нейросетей (StandardScaler использовался)
5.  Правильная обработка признаков (без утечки из будущего) критична

**Для улучшения можно попробовать:**

-   Attention механизмы
-   Transformer архитектуры (Temporal Fusion Transformer)
-   Ансамблирование моделей (Ridge + GRU)
-   Hyperparameter tuning (window_size, hidden_size, alpha для Ridge)
-   Feature engineering (дополнительные лаги, взаимодействие признаков)
-   Exogenous features (если доступны внешние данные)
